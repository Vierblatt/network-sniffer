# 网络嗅探器 — 项目完成度评估报告

## 一、项目概述

本项目实现了一个基于 Go 语言的**网络嗅探器（协议分析工具）**，包含三种运行形态：**CLI 命令行版**、**GUI 桌面版**（Windows 原生窗口）、**Web 版**（浏览器 HTTP 访问）。核心捕获引擎支持两种底层技术路径：**Windows 原始套接字**（`syscall.Socket` + `SIO_RCVALL`，无需 Npcap）和 **pcap/Npcap**。

## 四、三种构建方式

### 1. CLI 版（原始套接字，默认，**无需 Npcap**）

```bash
go build -o sniffer.exe .
```

- 使用 Windows 原始套接字捕获，捕获 IPv4 数据包
- 若原始套接字不可用（非 Windows 或权限不足），自动回退 pcap/Npcap
- 以管理员身份运行

### 2. GUI 桌面版（需先生成资源文件）

```bash
go install github.com/akavel/rsrc@latest
rsrc -manifest app.manifest -o rsrc.syso
go build -tags=gui -ldflags="-H windowsgui" -o sniffer_gui.exe .
```

- 使用 pcap/Npcap 捕获
- 功能：接口下拉选择、开始/停止/清空按钮、实时数据包表格（8 列）、分层详情面板（链路层→网络层→传输层→协议解析）、统计栏（总计/TCP/UDP/ICMP/ARP/速率）
- 以管理员身份运行

### 3. Web 版

```bash
go build -tags=web -o sniffer_web.exe .
```

- 启动后自动打开浏览器 → http://localhost:8080
- RESTful API + SPA 前端
- 实时数据包表格 + 分层详情 + 统计面板
- 以管理员身份运行

## 五、关键技术实现

### 5.1 原始套接字（核心要求）

```go
// capture/raw_socket.go — Windows 原始套接字捕获器

// 1. 初始化 Winsock
syscall.WSAStartup(0x202, &wsaData)

// 2. 创建原始套接字
socket, _ := syscall.Socket(syscall.AF_INET, syscall.SOCK_RAW, syscall.IPPROTO_IP)

// 3. 绑定到指定 IP
syscall.Bind(socket, &addr)

// 4. 启用混杂模式（接收所有流经网卡的数据包）
syscall.WSAIoctl(socket, SIO_RCVALL, ...)

// 5. 接收数据包
syscall.Recvfrom(socket, buf, 0)
```

原始套接字捕获的是 **IP 层数据包**（无以太网帧头），因此分析器提供了 `AnalyzeRaw()` 方法直接从 `data[0]` 解析 IPv4 头部。

### 5.2 协议分析

```
Ethernet 帧 (14B) → IPv4 头部 (20B+) → TCP/UDP/ICMP 段
     │                    │                  │
     ├─ dstMAC (0:6)     ├─ srcIP (12:16)   ├─ srcPort (0:2)
     ├─ srcMAC (6:12)    ├─ dstIP (16:20)   ├─ dstPort (2:4)
     └─ etherType (12:14)├─ protocol (9)    └─ flags/seq/win (TCP)
                         └─ totalLen (2:4)
```

- 端口 → 应用协议映射：HTTP(80)、HTTPS(443)、DNS(53)、SSH(22)、FTP(21)、SMTP(25)、MySQL(3306)、RDP(3389)、Redis(6379) 等 15+ 协议
- TCP 标志位：SYN(0x02)、ACK(0x10)、FIN(0x01)、RST(0x04)、PSH(0x08)

### 5.3 GUI 架构

- 基于 `lxn/walk` Windows 原生 GUI 工具包
- Model-View 分离：`PacketTableModel` / `IfaceListModel`
- 后台捕获协程 + 主线程定时刷新（400ms），通过 `packetsDirty` 标志避免无效刷新
- 三变量控制滚动行为：`autoScroll`（用户是否在底部）、`programmaticSelect`（区分代码/用户选择）、`packetsDirty`（有无新数据）

### 5.4 Web 架构

- 基于 Go 标准库 `net/http`
- RESTful API 设计（接口列表/开始/停止/清空/数据包/统计）
- SPA 前端（纯 JavaScript，无框架依赖）
- `Promise.all` 并发请求数据 + 统计，避免轮询竞态
- 智能自动滚动：仅在用户位于底部附近时滚动

## 八、注意事项

1. 所有版本均需**以管理员权限运行**
2. GUI 版编译前需执行 `rsrc` 命令生成 `rsrc.syso`
3. 原始套接字仅支持 IPv4 数据包捕获（Windows 限制）
4. pcap 版需安装 [Npcap](https://npcap.com) 驱动
5. Web 版启动后自动在默认浏览器中打开 http://localhost:8080
