# 网络嗅探器 — 构建与运行说明

## 开发环境

- **语言**：Go 1.26
- **OS**：Windows 11
- **依赖库**：gopacket v1.1.19（pcap 模式）、lxn/walk（GUI 模式）
- **驱动**：Npcap 1.79（pcap 模式需安装）

## 三种构建方式

### 1. CLI 版（原始套接字，默认）

```bash
go build -o sniffer.exe .
```

- 使用 Windows 原始套接字（`syscall.Socket` + `SIO_RCVALL`），**无需 Npcap**
- 若原始套接字不可用（非 Windows 或权限不足），自动回退到 pcap/Npcap
- 以管理员身份运行

### 2. GUI 桌面版（需先生成资源文件）

```bash
go install github.com/akavel/rsrc@latest
rsrc -manifest app.manifest -o rsrc.syso
go build -tags=gui -ldflags="-H windowsgui" -o sniffer_gui.exe .
```

- 使用 pcap/Npcap 捕获
- 以管理员身份运行
- 功能：接口下拉选择、开始/停止/清空按钮、实时数据包表格、分层详情面板、统计栏

### 3. Web 版

```bash
go build -tags=web -o sniffer_web.exe .
```

- 启动后自动打开浏览器 → http://localhost:8080
- RESTful API + SPA 前端
- 以管理员身份运行

## 注意事项

- 所有版本均需**以管理员权限运行**
- GUI 版编译前需执行 `rsrc` 命令生成 `rsrc.syso`
- pcap 版需安装 [Npcap](https://npcap.com)
- 原始套接字仅支持 IPv4 数据包捕获

## 项目结构

```
sniffer/
├── analyzer/analyzer.go      # 协议分析（Ethernet→IPv4→TCP/UDP/ICMP→应用层）
├── capture/
│   ├── capture.go            # pcap/Npcap 捕获器
│   └── raw_socket.go         # Windows 原始套接字捕获器
├── web/
│   ├── static/app.js         # Web 前端 JS
│   └── templates/index.html  # Web 前端页面
├── main.go                   # CLI 入口（原始套接字 + pcap 回退）
├── gui.go                    # GUI 入口（walk 桌面应用）
├── server.go                 # Web 入口（HTTP 服务器）
├── go.mod / go.sum           # Go 模块依赖
└── app.manifest              # Windows GUI 清单
```

## 与任务书要求的对应

| 任务书要求 | 实现方式 |
|-----------|---------|
| 2.1 原始套接字与网卡绑定 | `capture/raw_socket.go`：`syscall.Socket(AF_INET, SOCK_RAW, IPPROTO_IP)` + `SIO_RCVALL` |
| 2.2 源/目的 IP | `analyzer/analyzer.go` `parseIPv4()`：解析 IPv4 头部 data[12:16]、data[16:20] |
| 2.3 传输层协议+端口 | `parseTCP()`、`parseUDP()`：解析协议号 data[9]，端口号 data[0:2]/data[2:4] |
| 2.4 数据包长度 | `AnalyzeResult.Length`：IPv4 总长度字段 data[2:4] |
| 2.5 多层协议分析 | 链路层→网络层→传输层→应用层（加分项） |
| 2.6 显示分析结果 | CLI 实时输出 / GUI 表格+详情面板 / Web 可视化界面 |
